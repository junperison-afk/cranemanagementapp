# クレーン管理アプリ UI設計書（草案）

## 1. ガイドライン
- **トーン＆マナー**：落ち着いた業務アプリ向けのライトテーマ。余白を多めに取り、情報量が多い画面でも視認性を確保する。
- **カラーパレット**（添付UIを参考に抽出）
  - プライマリ：#3B63F6（ボタン、強調アクション）
  - セカンダリアクセント：#6573FF（ヘッダー、アクティブタブ）
  - サイドバー／テキストアイコン：#5B6C8F
  - サーフェス背景：#F4F6FB（コンテンツ背景）、#FFFFFF（カード・モーダル）
  - ステータス（成功系バッジ）：#43C17B
  - 境界線・デバイダー：#E1E6F0
- **タイポグラフィ**：サンセリフ系（Noto Sans / Source Sans）を基本とし、本文14px、見出し16-20px程度で階層化。
- **レスポンシブ方針**：PCを基準としつつ、幅 < 768px では1カラム化・サイドバー折りたたみ・テーブルのカード化を行う。
- **共通レイアウト**：一覧画面は共通リストビュー、詳細画面は共通レコードビューを利用し、コンポーネントを再利用する。
- **アクセシビリティ**：WCAG AA相当を目標にコントラスト比を確保し、主要アクションにはキーボードフォーカススタイルを適用する。
- **リファレンス**：Zoho CRM の画面構成・操作フローをベースとし、主要レイアウト／コンポーネントの挙動は Zoho CRM の体験に準拠させる。

## 3. 画面設計テンプレート

### 3.1 ログイン画面
- **画面ID**：UI-001
- **対象機能**：認証
- **利用ロール**：全ユーザー（管理者／編集者／閲覧者）
- **対応デバイス**：PC / Tablet / Mobile（レスポンシブ）

#### 3.1.1 レイアウトサマリ
- 中央寄せのログインカードを配置。背景は淡いグラデーションまたはブランドカラーの単色。
- カード内にアプリロゴ、フォーム（ID・パスワード）、ログインボタン、補助リンク（パスワードリセット等）を縦並び配置。
- モバイルではカード幅を画面幅の90%程度に縮小。

#### 3.1.2 コンポーネント詳細
- **フォーム**
  - `ユーザーID`（テキスト入力）
  - `パスワード`（パスワード入力）
- **アクション**
  - `ログイン` ボタン（主要アクション）
  - `パスワードをお忘れの方はこちら` テキストリンク（将来実装予定）

#### 3.1.3 ユースケース・シナリオ
- `ID` と `パスワード` を入力 → `ログイン` → 認証成功時にホーム画面（UI-010）へ遷移。
- 入力不足・誤入力時はフォーム上部にエラーメッセージ表示。

#### 3.1.4 バリデーション／エラーハンドリング
- 必須：ID・パスワードとも入力必須。空欄の場合は即時エラー表示。
- 認証失敗：汎用エラーメッセージ「IDまたはパスワードが正しくありません」。
- 連続失敗時のロックアウト等は別途検討。

#### 3.1.5 備考
- SP向けソフトキーボードが表示された際でもフォームが隠れないよう余白調整。
- 将来的なパスワードリセット・2段階認証導線はここに追加予定。

### 3.2 ホーム（ダッシュボード）
- **画面ID**：UI-010
- **対象機能**：プロジェクト進捗、営業案件状況のダッシュボード
- **利用ロール**：管理者／編集者（閲覧者は参照のみ）
- **対応デバイス**：PC（主）、Tablet（縦スクロール対応）、Mobile（カード表示へ簡略化）

#### 3.2.1 レイアウトサマリ
- 左側：共通サイドバー（全画面共通コンポーネント）固定表示。
- 右側：メインコンテンツ領域に2カラムカードレイアウト。
  - 上段：進行中プロジェクトのカード一覧（ステータス／次タスク／担当者）。
  - 下段：営業案件一覧（テーブルまたはカード、ステータス別フィルタ）。

#### 3.2.2 コンポーネント詳細
- **進行中プロジェクトカード**
  - プロジェクト名、顧客名、ステータス、次回点検日、担当者アイコン。
  - 詳細へ遷移する `詳細を見る` ボタン。
- **営業案件一覧**
  - テーブル形式（案件名、顧客名、ステータス、更新日、担当者）。
  - ステータスフィルタ（タブまたはプルダウン）。
  - 新規案件ボタン（右上配置、管理者／編集者のみ）。

#### 3.2.3 ユースケース・シナリオ
- ログイン直後に概要を確認。
- 進行中プロジェクトカードから詳細画面へ遷移。
- 営業案件一覧から該当案件を開き、進捗を更新。

#### 3.2.4 バリデーション／エラーハンドリング
- ダッシュボード自体に入力はなし。データ取得失敗時にはリトライボタンとエラーメッセージ表示。
- ロールによってボタン表示を切り替え（閲覧者は新規ボタン非表示）。

#### 3.2.5 備考
- カード・テーブルのデータは非同期で更新し、最新状況を反映。
- KPIウィジェットを将来的に追加する際のスペースを想定しておく。

### 3.3 共通UIコンポーネント（サイドバー）
- **役割**：全画面共通のメインナビゲーション。ログイン後の全ページで左側に固定表示。
- **構成**
  - 上部：アプリロゴ／システム名表示。
  - メニューリスト（上から順に）
    1. ホーム
    2. 営業案件一覧
    3. プロジェクト一覧
    4. 取引先一覧
    5. 連絡先一覧
    6. 機器一覧
    7. 設定
    8. ヘルプ
    9. ログアウト（下部に配置）
  - メニューにはアイコン＋ラベルを表示し、アクティブ項目をハイライト。
- **表示デバイス**
  - PC：常時固定表示（幅約240px）。
  - Tablet/Mobile：ヘッダーのメニューボタン押下でスライドイン（ハンバーガーメニュー）。
- **権限制御**
  - 権限に応じて項目表示を制御（例：設定は管理者のみ）。
  - ログアウトは全ユーザーが利用可能。
- **備考**
  - 折りたたみ（ミニモード）対応は今後の検討項目。
  - メニューの増減に備え、コンポーネントとして再利用可能な設計にする。

### 3.4 一覧画面共通レイアウト（リストビュー）
- **適用画面**：営業案件一覧、プロジェクト一覧、取引先一覧、連絡先一覧、機器一覧 など全てのリスト画面。
- **レイアウト構成**
  1. **ヘッダー／ツールバー**
     - 左側：フィルターメニュー（例：「すべての○○」プルダウン、検索フィールド）。
     - 右側：主要アクション（例：`+ 新規作成` ボタン、その他ドロップダウン）。
  2. **データテーブル**
     - 先頭列：行選択チェックボックス。
     - 主要カラム（会社名、担当者、ステータス、連絡先など）は固定幅＋適切な余白。
     - ステータスなど重要情報はバッジ表示（例：緑色の「フォロー中」）。
     - 右端に詳細表示／アクションメニューアイコン。
  3. **フッター**
     - ページネーション（10/20/50件などの表示件数切替、ページ送り）。
- **スタイル指針**
  - 行ホバー時に背景色を薄く変更し、選択感を出す。
  - カラムヘッダーはソート可能（矢印アイコン表示）。
  - テーブル背景は淡色、境界線は最小限に抑え視認性を確保。
- **レスポンシブ対応**
  - Tablet：横スクロールを許容しつつ、最重要カラムが常に表示されるよう固定。
  - Mobile：カード型レイアウトに自動切替（主要項目のみ表示、詳細はタップで展開）。
- **共通アクション**
  - 行選択→一括操作（削除／エクスポート等、将来的に追加）。
  - アクションボタンから新規登録・インポート画面へ遷移。
  - 詳細画面への遷移は行クリックまたは末尾の「詳細」ボタン。
- **備考**
  - 添付画像を参考に UI トーンと余白を揃える。
  - 一覧画面ごとに追加フィルタ項目はヘッダー右側に追加しても統一感を維持。

### 3.5 詳細画面共通レイアウト（レコードビュー）
- **適用画面**：営業案件詳細、プロジェクト詳細、取引先詳細、連絡先詳細、機器詳細 など。
- **レイアウト構成**
  1. **ヘッダー**
     - 左上：戻るボタン（例：「一覧へ戻る」）とレコード名（案件名など）、関連取引先名を表示。
     - 右上：主要アクションボタン（例：`メールを送信する`、`次のステップへ`、`編集する`）とその他メニュー。
     - 最新更新日時を右上に表示。
  2. **タブ領域**
     - `概要` / `履歴` / `関連` など、対象に応じたタブを横並びで配置。デフォルトは `概要`。
  3. **詳細情報カード**
     - 中央のメインパネルに2カラムの情報テーブルを配置。
     - 左カラム：主要属性（ステータス、担当者、連絡先など）、右カラム：補足情報（メール、売上、業界等）。
     - セクションタイトル（例：基本情報／住所情報）で区切り、項目ラベルと値を横並びに整列。
     - 編集可能項目にはペンアイコンを表示し、インライン編集またはモーダル遷移を想定。
  4. **関連リストパネル**
     - 左側に縦型メニューを配置し、メモ／活動／添付／関連案件などを切替可能。
     - 選択した関連リストは下部または別タブに展開。
- **スタイル指針**
  - カード背景はホワイト、枠線・区切りは淡いグレーで統一。
  - 目立たせたいステータスはバッジ表示（色分け）。
  - 全体の余白は一覧画面と統一し、視覚的一貫性を保つ。
- **レスポンシブ対応**
  - Tablet：2カラムを1カラムに落とし、関連リストパネルは折りたたみ可能に。
  - Mobile：タブを横スクロールできるピル型ボタンにし、詳細情報は縦に積む。
- **共通アクション**
  - 戻るボタンで一覧へ戻る。
  - 編集ボタンで編集画面／モードへ遷移。
  - 右上のその他メニュー（縦三点）にエクスポート・削除等の副次アクションを配置。
- **備考**
  - 履歴タブではタイムライン表示を想定（作業ログ、コメント等）。
  - 関連リストの順番や表示項目は画面種別によって調整可能だが、レイアウトは共通化する。
